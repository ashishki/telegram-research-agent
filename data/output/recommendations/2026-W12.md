## 2. Temporal Consistency Techniques in Diffusion-Based Video Models

**Why now:** Following from the same 4-week signal — once you can call a video API, the immediate practical problem is frame-to-frame coherence degradation. This is where most production workflows break down.

**What to study:** Study **attention-based temporal conditioning** — specifically how models use causal attention masks across frame tokens vs. full 3D attention. Compare the architectural tradeoff between AnimateDiff-style motion modules (bolt-on) and native spatiotemporal models (HunyuanVideo, CogVideoX). Understand what "noise schedule consistency" means across the temporal axis.

**Concrete action:** Run two generations of the same 3-second clip — one with a fixed seed and one with varied per-frame seeds — and measure structural similarity (SSIM) between consecutive frames using `scikit-image`. Log results in your `OpenClaw_Evaluation_Lab` project, which appears purpose-built for this kind of comparative eval work.

## 3. Containerized Video Generation Workloads with GPU Scheduling

**Why now:** You have a strong Dockerfile pattern across at least 10 projects, plus `gensyn-node-watchdog` with explicit GPU-first support. Video generation models have severe VRAM requirements that require explicit resource management strategies.

**What to study:** Learn **CUDA memory fraction pinning and multi-process GPU scheduling** in Docker — specifically `--gpus` device isolation flags, `PYTORCH_CUDA_ALLOC_CONF=max_split_size_mb`, and how to use `nvidia-smi` MIG (Multi-Instance GPU) partitioning for serving multiple inference requests without OOM crashes. Compare this with quantized inference via `bitsandbytes` INT8 for video UNets.

**Concrete action:** Extend the pattern from `gensyn-docker-multi` to create a video inference container that exposes a `/generate` HTTP endpoint and handles graceful VRAM cleanup between requests using `torch.cuda.empty_cache()` with explicit model offloading.

## 5. Prompt Templating Architectures for Multi-Modal Content Pipelines

**Why now:** You use Jinja in both `gdev-content` and `telegram-research-agent`, and you're tracking video generation tools deeply. The missing link is a systematic approach to prompts that span text → image → video across a single content pipeline.

**What to study:** Examine **prompt inheritance and slot-filling patterns** in multi-modal systems — specifically how to define a base narrative template that derives video scene prompts, thumbnail image prompts, and SEO meta descriptions from a single source-of-truth context object. Look at how LangChain's `ChatPromptTemplate` with partial variable binding differs from raw Jinja for this use case.

**Concrete action:** In `gdev-content`, design a `ContentManifest` Pydantic model that holds a topic, tone, and key entities, then write three Jinja templates that each derive a different modal artifact (video prompt, article intro, meta description) from the same manifest instance. This directly reduces prompt drift across your content toolchain.
